# 在Global Admin Console中更新組織原則

**套用至：**&#x200B;企業

瞭解全域管理員如何在Global Admin Console中為組織及其子系設定和修改原則。

>[!NOTE]
>
>在[Global Admin Console](https://helpx.adobe.com/enterprise/global-admin-console/adopt-global-administration.html)中，從階層選取組織，並導覽至&#x200B;**原則**&#x200B;索引標籤以允許或禁止或鎖定原則。
>
> [登入Global Admin Console](https://global-admin-console.adobe.com/)

原則與組織相關聯，並限制可在該組織上執行的作業。 設定原則值時，會限制或啟用此後任何動作。
例如，如果**宣告網域**&#x200B;原則設為&#x200B;*不允許*，則無法宣告其他網域，但在設定原則值之前宣告的任何網域都不會受到影響。

## 設定原則

若要修改組織的原則，請執行下列動作：

1. 在Global Admin Console中，[選取要編輯的組織](https://helpx.adobe.com/enterprise/global-admin-console/overview.html)，然後導覽至&#x200B;**[!UICONTROL 原則]**&#x200B;索引標籤。
1. 為相關原則選取切換以允許或不允許它。 您也可以鎖定原則，讓[選取的組織](https://helpx.adobe.com/enterprise/global-admin-console/overview.html)或其上層組織的全域管理員以外的人無法變更或解除鎖定原則。
1. 若要鎖定原則，請選取&#x200B;**[!UICONTROL 鎖定]** ![鎖定](./assets/lock.png)圖示。 將游標暫留在鎖定上會顯示所選組織的名稱。 深入瞭解[原則鎖定](#policy-locks)。
1. 在您編輯完組織後，選取&#x200B;**[!UICONTROL 檢閱擱置中的變更]**。 檢閱後，選取&#x200B;**[!UICONTROL 提交變更]**&#x200B;以[執行](https://helpx.adobe.com/enterprise/global-admin-console/execute-jobs.html)它們。

## 原則鎖定 {#policy-locks}

鎖定原則時，除非解除鎖定原則，否則無法變更其值。 Global Admin Console記憶組織選擇器中的[選取的組織](https://helpx.adobe.com/enterprise/global-admin-console/overview.html)為鎖定原則的組織。 所選組織或樹狀結構中較高組織的任何全域管理員都有權解鎖原則。 範圍小於該組織的全域管理員沒有解除鎖定和變更原則值的許可權。

若要建立鎖定的環境，請在子組織上設定所需的原則值，然後加以鎖定。 這些子組織的全域管理員將無法編輯原則值。

### 範例：鎖定的環境

如果&#x200B;*Acme Division*&#x200B;的全域管理員Elissa建立子組織&#x200B;*行銷*&#x200B;和&#x200B;*工程*，則將Robert新增為&#x200B;*行銷*&#x200B;的全域管理員，並將Sarah新增為&#x200B;*工程*&#x200B;的全域管理員。 接著，她將數個原則設定為&#x200B;*不允許*&#x200B;及&#x200B;*鎖定*。 當Elissa選擇&#x200B;*Acme部門*&#x200B;作為選取的組織時，她可以稍後解除鎖定並變更原則值，但Robert和Sarah無法解除鎖定他們身為全域管理員之組織上的原則，因為原則已由組織&#x200B;*Acme部門*&#x200B;鎖定。

## 原則詳細資料

### 組織管理

| 原則名稱 | 說明 |
| --- | --- |
| **建立子組織** | 允許全域管理員建立子組織。 如果&#x200B;*off*，則無法建立任何子組織。 |
| **重新命名組織** | 如果允許，全域或系統管理員可以重新命名組織。 它也能控制組織國家/地區的變更。 如果重新命名父級組織，或重新命名組織或組織的祖項，則也可以獨立於此原則設定變更組織的路徑名稱。 |
| **刪除組織** | 允許全域管理員刪除子組織。 當具有企業儲存空間的組織因為有刪除使用者資產的風險而啟用時，這一點就變得更重要。 |

### 管理員管理

| 原則名稱 | 說明 |
| --- | --- |
| **新增或刪除管理員** | 允許全域管理員新增管理員至組織。 如果&#x200B;*off*，則無法新增管理員。 |
| 建立子組織時&#x200B;**從父項繼承系統管理員** | 當全域管理員建立新的子組織時，父系的系統管理員會自動成為新組織的系統管理員。 此原則預設為&#x200B;*off*。 |
| **管理管理員** | 允許全域管理員變更或移除/編輯管理員許可權。 |

### 使用者管理

| 原則名稱 | 說明 |
| --- | --- |
| **從父級組織管理的目錄繼承使用者** | 在建立新的子組織之前，此原則必須在&#x200B;*上切換*&#x200B;並啟用。 建立子組織時，父組織中的使用者可作為子組織中的使用者。 換言之，在GAC內建立新子系時，此原則會自動設定父項與子項之間的信任關係。 對於現有組織，加入GAC之前的任何信任關係在加入GAC後都將保留。 如果沒有信任關係，則必須遵循一般的信任請求流程。 若要讓此原則成功，建立新組織的全域管理員也必須是具有已宣告網域的父級組織的系統管理員。 如果沒有，網域信任關係將不會繼承到新建立的組織。 |
| **新增Adobe ID使用者** | 如果設定，組織將無法透過Admin Console、User Management API (UMAPI)或同步機制新增Adobe ID型別使用者。 |
| **管理使用者群組** | 如果允許，全域、系統和使用者群組管理員可以建立、編輯和刪除使用者群組。 |

### 目錄和網域強制執行

| 原則名稱 | 說明 |
| --- | --- |
| **宣告網域** | 如果設定，系統管理員可以在Admin Console上宣告網域。 |
| **變更身分設定** | 如果設定，系統管理員可以在Admin Console上變更使用者身分設定的設定。 |

### 產品配置

| 原則名稱 | 說明 |
| --- | --- |
| **管理產品** | 允許全域管理員新增或移除產品，以及變更產品資源授權。 |

### 資產共用

| 原則名稱 | 說明 |
| --- | --- |
| **系統或存放裝置管理員可以變更資產共用設定** | 如果允許，存放裝置和系統管理員可以變更資產共用設定，包括安全性連絡人、密碼原則以及存放裝置原則。 |
| 建立組織時&#x200B;**從父系繼承共用原則** | 如果允許，在建立子組織時，資產共用設定會繼承自父項。 資產共用設定包含安全性連絡人、密碼原則及儲存原則。 這僅適用於建立時新建立的組織。 它會設定在父項上，並影響在該父項下建立子組織。 |
